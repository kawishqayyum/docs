## 地理位置定位
<Tip>
  Common setup is to create a zone per each location, this way you can control your requests target on the zone level. 
 
  For example: ispshared-uk, ispshared-us, ispshared-de. 
</Tip>

### Dedicated and Pay per IP proxies
地理位置定位 allows you to target specific `Country`.
Select the prefered countries from the drop-down menu. 

If you select multiple countries, we assign proxies distributed  evenly across the countries you selected.

<Frame>
  <img src="/images/proxy-networks/data-center/geotargeting.png" alt='地理位置定位'/>
</Frame>

### 共享 proxy
In 互联网服务提供商 (ISP) 共享 proxies, you have access to 20,000 proxies around the globe. To target specific country, use the flag -country in the proxy user name parameter with an ISO-3166 country code. 

[FAQ: Where can I see the list of country codes?](https://docs.brightdata.com/cn/general/faqs#where-can-i-see-the-list-of-country-codes)

## 缓存代理
Bright Data's cache proxy is a feature that aims to reduce the unneeded load from target sites by allowing customers to have their successful results cached on the one hand and, on the other hand, to receive cached results to their requests when possible.
<AccordionGroup>
  <Accordion title="更健康的生态系统">
    从技术上讲，通过消耗网站资源并在数据收集领域造成性能问题可能使网站过载。 Bright Data 除了安装系统以避免目标站点过载之外，还提供缓存代理功能，以避免在我们从另一位客户提出的相同请求中获得最新缓存记录的情况下，向目标站点提出冗余请求。
  </Accordion>

  <Accordion title="提高了性能">
    从我们的缓存中收到的响应速度大约快了 20 倍！
    This means that as our cache usage grows, performance will keep increasing.
  </Accordion>

  <Accordion title="经济利益">
    为了促进我们维护健康生态系统的目标，对于通过选择使用缓存代理的代理区域中的任何使用量，我们将提供使用折扣。

    虽然我们将提供的缓存记录可能只占代理区域总使用量的一小部分，但折扣将充分适用于代理区域。
  </Accordion>

</AccordionGroup>

## 高级选项：

### 100% uptime
The 100% uptime brings immediate value by providing 100% connectivity and continuous high-performance level, free of charge. Plus you don't have to make any changes in your code/workflow.

If our system detects a problem, like a connectivity issue or an IP where the GEO isn't exactly what you asked for when buying that IP, we will automatically route your requests through other IPs which are exactly the same as the original IPs.

<Accordion title="Seamless integration">
  To provide our customers a seamless experience, so we made some exceptions:
  * We will not assign a fallback IP if you have specified an IP in your request
  * 100% uptime will not interrupt a live connection. If the fallback is needed, it will play in once the next connection is established
</Accordion>

### Enable 100% uptime
Simply toggle the 100% uptime switch
<Frame>
    <img src="/images/proxy-networks/data-center/100uptime.png" alt='100% uptime'/>
</Frame>

<Tip>You can also turn on/off 100% uptime feature via the API.</Tip>

### 端口
端口 `80` and `443` are available by default, supporting HTTP and HTTPS protocols.
The below table shows supported protocols by port:

| Port   | Protocol              |
|--------|-----------------------|
| `80`   | HTTP and HTTP CONNECT |
| `443`  | HTTPS                 |

<Accordion title="Request Additional ports">
  Bright Data can support additional ports by request. A dedicated and additional compliance process with the Bright Data compliance team will follow every request to support a new port.
  If you would like additional port permissions, you can contact Bright Data Support.
  Examples of ports that require Bright Data compliance review before activation:
  | Port   | Protocol |
  |--------|----------|
  | `8443` | HTTP     |
  | `8243` | HTTPS    |
</Accordion>
